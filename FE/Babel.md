# Babel이란?

> **Babel은 최신 JavaScript(ES6+) 문법을 이전 버전(ES5)으로 변환해주는
> 트랜스파일러(Transpiler)이다.**\
> 이를 통해 최신 문법으로 작성한 코드가 **구형 브라우저에서도 동작**할
> 수 있게 도와준다.

---

## 왜 Babel이 필요한가?

JavaScript는 ECMAScript 표준을 기반으로 발전해 왔다.\
특히 ES6 이후 많은 기능이 추가되었지만, **모든 브라우저가 최신 문법을
지원하는 것은 아니다.**

-   최신 브라우저 → ES6+ 잘 지원\
-   구형 브라우저 → 최신 문법을 이해하지 못해 오류 발생

결과적으로 **사용자가 구형 브라우저를 사용하면 화면이 제대로 동작하지
않는 문제가 발생**했다.\
이 문제를 해결하기 위해 등장한 도구가 바로 **Babel**이다.

---

## Babel의 탄생 배경

초기에는 ES6 문법을 ES5로 변환하는 프로젝트였던 **6to5**가 존재했다.\
이후 더 확장된 기능을 제공하며 **Babel**이라는 이름으로 재탄생했다.

---

# Babel 동작 원리

Babel은 코드를 변환하는 과정이 총 **3단계**로 이루어진다.

---

## 1. 파싱(Parsing)

-   원본 JavaScript 코드를 읽는다.
-   읽은 코드 구조를 분석하여 **AST(Abstract Syntax Tree)** 로 변환한다.

---

## 2. 변환(Transformation)

-   생성된 AST를 **정해진 규칙(Rule)** 에 따라 변환한다.
-   이 규칙들이 Babel에서는 **플러그인(Plugin)** 으로 제공된다.

---

## 3. 출력(Printing)

-   변환이 끝난 AST를 다시 **JavaScript 코드 문자열로 변환**한다.

---

# Babel Plugin (플러그인)

Babel이 변환 작업을 할 때 사용하는 **규칙의 묶음**이다.

---

# Babel Preset (프리셋)

여러 플러그인을 묶어 제공하는 기능이며, 대표적으로 **@babel/preset-env**
가 있다.

---

# 정리

| 항목      | 설명                               |
|-----------|------------------------------------|
| Babel     | 최신 JS → 구버전 JS로 변환하는 도구 |
| 필요 이유 | 구형 브라우저 지원                 |
| 동작 단계 | 파싱 → 변환 → 출력                 |
| 플러그인  | 문법 변환 규칙                      |
| 프리셋    | 여러 플러그인 묶음                  |
